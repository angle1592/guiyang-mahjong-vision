# 贵阳捉鸡麻将实时识牌助手设计

日期：2026-06-30

## 目标

为当前 Windows 电脑、当前微信小程序窗口尺寸和当前牌面素材开发一个低延迟本地助手。程序实时识别玩家底部手牌，在置顶小窗中显示识别结果和推荐弃牌。

第一版只提供建议，不自动点击或代打。

## 成功标准

- 在保存的、来自当前小程序的标注回放截图上，单牌识别准确率不低于 99%。
- 单帧“截取、识别、建议”处理耗时的第 95 百分位低于 100 ms。
- 只有在 14 张牌均被可靠识别且画面稳定时才显示弃牌建议。
- 低置信度、窗口不可见、牌数异常或动画状态下不输出误导性建议。

## 范围

### 第一版包含

- 按窗口标题定位当前微信麻将小程序。
- 按固定坐标截取玩家底部 14 个牌槽。
- 使用 OpenCV 模板匹配识别手牌。
- 通过标注模式逐步收集当前牌面素材的模板。
- 在置顶小窗显示手牌、建议弃牌、识别置信度和处理耗时。
- 基于手牌效率与贵阳捉鸡规则偏好生成弃牌建议。
- 保存标注截图，并通过离线回放验证识别准确率和性能。

### 第一版不包含

- 自动点击、自动碰杠胡或任何公开对局代打功能。
- 对手牌河、副露和剩余牌的完整分析。
- 对其他窗口尺寸、显示缩放、主题或牌面皮肤的适配。
- 神经网络训练或通用目标检测。

## 技术方案

采用“固定牌槽裁剪 + OpenCV 模板匹配”。当前目标环境中的窗口尺寸和牌面固定，因此该方案比通用 OCR、目标检测或视觉语言模型更快、更容易验证。

每个牌槽同时计算：

- 灰度图的归一化模板匹配分数；
- 边缘图的归一化模板匹配分数。

两项分数按固定权重合并。边缘分数降低亮度、高亮和轻微阴影变化的影响，灰度分数保留牌面细节。模板与牌槽使用一致尺寸，不进行全屏滑窗搜索。

## 组件

### `capture.py`

- 通过窗口标题定位“多乐贵阳捉鸡麻将”。
- 获取窗口位置并按配置裁剪手牌区域。
- 输出 14 个固定尺寸的牌槽图像。
- 检查窗口是否存在、是否最小化以及目标区域是否在屏幕内。

第一版要求小程序窗口保持可见且不被其他窗口遮挡。

### `templates.py`

- 管理 `templates/<tile-name>/` 下的多个样本。
- 对模板和待识别图像执行一致的裁剪、缩放、灰度化和边缘化。
- 返回最佳牌名、最佳分数和次佳分数。
- 同时使用绝对阈值和“最佳与次佳分差”判断置信度。

### `calibration.py`

- 展示未识别或低置信度牌槽的缩略图。
- 允许用户从 34 种数牌中选择标签。
- 保存原始裁剪图、规范化模板和元数据。
- 同一张牌允许保存普通、选中、高亮等多个状态样本。

### `recognizer.py`

- 识别 14 个牌槽。
- 输出牌名列表、逐牌置信度、未知牌位置和处理耗时。
- 连续两帧识别结果一致后，才将结果标记为稳定。

### `advisor.py`

对每一种可弃牌方案计算：

1. 胡牌距离；
2. 有效进张种类和理论剩余张数；
3. 牌形质量；
4. 贵阳规则偏好。

第一版的贵阳规则偏好包括：

- 对幺鸡和八筒给予可配置的保留权重；
- 因不能吃牌，适度提高对子及潜在刻子的价值；
- 胡牌距离优先于鸡牌保留，避免为扣鸡显著拖慢听牌；
- 不进行基于对手牌河的危险度判断。

建议结果包含推荐弃牌和简短理由，例如“打九万：保持五组牌结构，有效进张最多”。

### `overlay.py`

使用轻量 Windows 置顶小窗显示：

- 当前识别手牌；
- 推荐弃牌；
- 简短理由；
- 最低识别置信度；
- 本帧处理耗时；
- 当前状态：识别中、等待稳定、需要标注或窗口不可用。

小窗放置在手牌区域之外，避免遮挡截图目标。

### `app.py`

- 启动各组件并维护约 20 FPS 的处理循环。
- 对重复稳定画面复用上一次建议，避免重复计算。
- 捕获异常并更新小窗状态，不因单帧失败退出。

## 数据流

```text
微信小程序窗口
    -> 固定区域截图
    -> 14 个牌槽裁剪
    -> 灰度与边缘模板匹配
    -> 稳定性及置信度检查
    -> 手牌效率与贵阳权重评分
    -> 置顶小窗显示
```

当任意牌低置信度时，流程在建议生成前停止，并将对应牌槽送入标注模式。标注完成后，后续帧立即使用新增模板。

## 配置与数据

项目使用独立目录 `guiyang-mahjong-vision/`，其中：

```text
guiyang-mahjong-vision/
  src/
  tests/
  templates/
  samples/
  config.json
  requirements.txt
  README.md
```

`config.json` 保存窗口标题、手牌区域坐标、牌槽尺寸、匹配阈值、帧率和鸡牌权重。模板及样本不写入源代码。

## 错误处理

- 找不到窗口：小窗显示“等待麻将窗口”。
- 窗口最小化或目标区域不可见：停止建议。
- 牌槽为空或识别数量不是 14：等待下一帧。
- 最佳分数低于阈值，或最佳与次佳分差过小：进入标注状态。
- 连续帧识别不一致：显示“动画中”，不更新建议。
- 决策器遇到非法手牌：记录诊断信息并停止建议。

## 测试方案

### 单元测试

- 模板预处理输出尺寸稳定。
- 已知模板能够匹配到正确牌名。
- 相似牌的最佳与次佳分差能够触发低置信度保护。
- 合法与非法手牌校验。
- 已知手牌的胡牌距离、有效进张和弃牌排序。
- 幺鸡、八筒权重不会覆盖明显更优的胡牌距离。

### 回放测试

- 从真实小程序收集并人工标注截图。
- 离线逐帧运行完整识别流程。
- 输出单牌准确率、完整手牌准确率、未知率和延迟分布。
- 以保存样本运行测试，不依赖在线牌局。

### 手动验收

- 启动小程序和助手。
- 完成若干首次模板标注。
- 观察稳定手牌是否在 100 ms 内更新。
- 验证动画、托管遮罩和窗口切换时不会输出错误建议。

## 实施边界

识别和建议完全在本地执行。第一版不会自动输入、发送消息、充值、分享或修改游戏账户状态。
